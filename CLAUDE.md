# Calendar Project Guidelines

## Project Overview
A simple calendar application with date and time picker functionality, built with vanilla HTML, CSS, and JavaScript.

## Build/Run Commands
- Open index.html in a browser to run the application
- No build step required (static HTML/CSS/JS)

## Code Style Guidelines

### JavaScript
- Use camelCase for variables and functions
- Organize code in modular objects with clear responsibility
- Follow the pattern of separating elements, state, and methods
- Use ES6+ features (arrow functions, template literals)

### HTML/CSS
- Use descriptive class names with kebab-case
- Keep CSS selectors specific but not too nested
- Use CSS Grid and Flexbox for layouts
- Maintain mobile-first responsive design

### Naming Conventions
- HTML classes should be descriptive of their purpose
- JavaScript functions should be named according to their action
- Variables should clearly indicate what they contain

### Error Handling
- Use validation for user input
- Provide clear feedback for invalid date formats
- Gracefully handle edge cases with date parsing
